# Jira 深度分析报告: [KAN-2] [SV][SSD1250][Sanitize] Block Erase 期间下发 NVM Reset 导致下次上电设备进入只读模式

## 📋 问题摘要

**客户名称**: Micron
**测试项目**: SSD1250, Sanitize
**测试平台**: Platform: AMD Ryzen 7000 Series reference platform, OS: Ubuntu 22.04 LTS, Form Factor: M.2 2280

**测试步骤**:
1. 发起 Sanitize (Block Erase) 操作 `nvme sanitize /dev/nvme0 -a 2`
2. 轮询 Sanitize Status (Log Page 81h)，在进度达到 30%-50% 期间，通过 PCIe 写入 Controller Configuration (CC.EN=0 -> 1) 触发 NVM Reset
3. 等待设备重新枚举并 Ready
4. 尝试进行 4K Random Write

**根因**: 固件未实现 CSTS.RDY 等待机制
**修复方案**: 在 Sanitize Block Erase 阶段检测到 NVM Reset 时，先完成当前 block 的 erase，再进入 reset handler。预计性能影响 <2%。
