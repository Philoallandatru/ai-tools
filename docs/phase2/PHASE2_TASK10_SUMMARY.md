## 📋 任务目标

根据 REPORT_REVIEW.md 的评审反馈，评论分析存在以下问题：

1. **过于冗长**: 6 条评论分析占用了 80 行（约 38% 的报告长度）
2. **重复套话**: 每条评论都有相同的结构标签（时间线位置、关键决策、合理性评估）
3. **信息密度低**: 很多分析是套话，实际有价值的信息很少

**目标**: 减少 50% 篇幅，提高信息密度，同时保留核心价值。

## 🔍 问题分析

### 原有问题

根据 REPORT_REVIEW.md 第 104-156 行的评审意见：

```
#### 4. 评论分析过于冗长和重复

**问题示例**:

每条评论都有相同的结构：
```
时间线位置：该评论处于XXX阶段...
关键决策：包含/不包含...
合理性评估：...
```

**问题**:
- 6 条评论分析占用了 80 行（约 38% 的报告长度）
- 很多分析是套话，如"时间线位置"、"关键决策"等标签重复出现
- 实际有价值的信息很少
```

### 原始格式示例

```
**评论 #1**:
时间线位置：该评论处于问题发现阶段，是问题解决过程的起点。
关键决策：不包含具体的技术决策，主要是问题现象的描述和初步假设。
合理性评估：该评论合理地指出了现场与实验室环境的差异，为后续的根因分析提供了方向。
```

**问题**:
- 长度约 120 字符
- 包含大量套话（"该评论处于"、"是问题解决过程的起点"）
- 三个标签重复出现，占用空间
- 信息密度低

## 💡 解决方案

### 方案设计

创建 `CompactCommentAnalyzer`，采用精简格式和时间线视图相结合的方案。

**设计原则**:
1. **精简格式**: 每条评论最多 3 行（角色 + 摘要 + 关键点）
2. **去除套话**: 移除"时间线位置"、"合理性评估"等冗长标签
3. **保留核心**: 保留角色、内容摘要和关键价值点
4. **时间线视图**: 按阶段组织评论，清晰展示问题解决过程

### 精简格式设计

**格式**:
```
**评论 #N** [角色 - 姓名]:
评论摘要（1 句话概括评论内容）
→ 关键点（问题发现/技术决策/修复实施/效果验证/建议改进等，1 句话）
```

**示例**:
```
**评论 #1** [SV - Li Qiang]:
现场失败率 12%，实验室 0.5%，怀疑 PCIe 链路质量问题。
→ 问题发现：现场与实验室失败率差异大
```

**优势**:
- 长度约 70 字符（减少 42%）
- 去除了所有套话
- 使用 → 符号，更直观
- 保留了所有核心信息

### 时间线视图设计

**格式**:
```
问题发现:
  2026-05-02 08:00 - [SV] 现场失败率 12%，实验室 0.5%

方案讨论:
  2026-05-02 10:00 - [HW] 建议 pre-flight check (BER > 1e-12)
  
修复实施:
  2026-05-02 14:00 - [FW] 实现 pre-flight check 机制
  
效果验证:
  2026-05-02 16:00 - [FAE] 失败率降至 3%（剩余为断电）
  2026-05-02 18:00 - [DV] 1000 台测试，成功率 99.2%
```

**优势**:
- 按时间顺序组织，清晰展示问题解决过程
- 按阶段分组（问题发现、方案讨论、修复实施、效果验证、后续改进）
- 便于快速了解问题演进

## 🛠️ 实现细节

### 1. 创建 CompactCommentAnalyzer

**文件**: `crawler/analyzers/compact_comment_analyzer.py`

**核心功能**:
```python
class CompactCommentAnalyzer(ConfigurableAnalyzer):
    """精简评论分析器 - 用简洁格式分析评论"""
    
    def analyze(self, jira_data: Dict[str, Any], context: AnalysisContext) -> Dict[str, Any]:
        # 1. 构建精简提示词
        prompt = self._build_compact_prompt(comments_to_analyze, jira_data)
        
        # 2. 调用 LLM 一次性分析所有评论
        response = self.call_llm(prompt, context, default_max_tokens=2000)
        
        # 3. 解析响应为精简格式
        result = self._parse_response(response, comments_to_analyze)
        
        return result
```

### 2. Prompt 设计

**关键要素**:
- 明确要求精简格式（每条评论最多 3 行）
- 要求提取角色和姓名
- 要求用 → 标注关键点
- 避免冗长的分析和套话

**Prompt 示例**:
```
请为每条评论生成精简分析，格式如下：

**评论 #N** [角色 - 姓名]:
评论摘要（1 句话概括评论内容）
→ 关键点（问题发现/技术决策/修复实施/效果验证/建议改进等，1 句话）

**要求**:
1. 每条评论最多 3 行（标题 + 摘要 + 关键点）
2. 摘要要简洁，提取核心信息
3. 关键点用 → 开头，标注评论的价值
4. 如果评论没有实质内容，可以省略关键点
5. 从评论中提取角色和姓名（如 [SV - Li Qiang]）

要求：
- 去除冗长的分析和套话
- 只保留有价值的信息
```

### 3. 响应解析

**解析策略**:
1. 按 `**评论 #N**` 分割每条评论
2. 提取角色（从 `[...]` 中）
3. 提取摘要（第一行非空行）
4. 提取关键点（以 `→` 开头的行）

**核心方法**:
```python
def _parse_response(self, response: str, comments: List[str]) -> Dict[str, Any]:
    compact_analyses = []
    
    # 按 **评论 #N** 分割
    pattern = r'\*\*评论 #(\d+)\*\*\s*(.+?)(?=\*\*评论 #|\Z)'
    matches = re.finditer(pattern, response, re.DOTALL)
    
    for match in matches:
        index = int(match.group(1))
        content = match.group(2).strip()
        
        # 解析角色
        role_match = re.search(r'\[([^\]]+)\]', content)
        role = role_match.group(1) if role_match else '未知'
        
        # 提取摘要和关键点
        lines = content.split('\n')
        summary = ''
        key_point = ''
        
        for line in lines:
            line = line.strip()
            if line.startswith('→'):
                key_point = line[1:].strip()
            elif not summary and line and not line.startswith('['):
                summary = line
        
        compact_analyses.append({
            'index': index,
            'role': role,
            'summary': summary,
            'key_point': key_point
        })
    
    # 生成时间线视图
    timeline_view = self._generate_timeline_view(compact_analyses)
    
    return {
        'compact_analysis': compact_analyses,
        'timeline_view': timeline_view
    }
```

### 4. 时间线视图生成

**生成策略**:
1. 根据关键点中的关键词分类到阶段
2. 提取时间信息（如果有）
3. 按阶段组织输出
